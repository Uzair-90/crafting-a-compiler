### Question No. 9

Most compilers can produce a source listing of the program being com-
piled. This listing is usually just a copy of the source ﬁle, perhaps
embellished with line numbers and page breaks. Assume you are to
produce a prettyprinted listing.
(a) How would you modify a Lex scanner speciﬁcation to produce a
prettyprinted listing?
(b) How are compiler diagnostics and line numbering complicated
when a prettyprinted listing is produced?

### Solution

Exercise 9 asks how to generate a nicely formatted source listing while scanning. The scanner must print source lines with line numbers and align error messages correctly.

### Answer

The scanner should maintain:

```txt
1. Current line number
2. Current column position
3. A buffer containing the current source line
```

Algorithm:

```txt
When a character is read:
    append it to current line buffer
    increment column counter

When newline is encountered:
    print line number + buffered source line
    clear buffer
    increment line number
    reset column counter
```

For lexical errors:

```txt
1. Print the source line
2. Print an indicator (^ or ****) under the offending token
3. Print an explanatory error message
```

Example:

```txt id="p6k6g5"
15  bal = 1.0e-n;
                ^
Lexical Error: malformed floating literal
```

Important implementation detail:

```txt
The scanner should not immediately discard characters after reading them.
It should preserve enough of the line to reproduce source listings and diagnostics.
```

Benefits:

```txt
- Easier debugging
- Better compiler diagnostics
- Precise error locations
- Useful for batch compilation systems
```
