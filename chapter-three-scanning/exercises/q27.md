### Question No. 27 

Let F be any NFA that contains λtransitions. Write an algorithm that
transforms F into an equivalent NFA F that contains no λ transitions.
Note: You need not use the subset construction, since you are creating an
NFA, not a DFA.

### Exercise 27

Goal: remove all **λ-transitions** from an NFA while keeping the same language.

A λ-transition is a transition that consumes **no input**.

---

### Construction

Suppose the NFA is:

```txt
N = (Q, Σ, δ, q0, F)
```

Create a new NFA:

```txt
N' = (Q, Σ, δ', q0, F')
```

with **no λ-transitions**.

---

### Step 1: Compute λ-closure

For every state `q`, compute:

```txt
λ-closure(q)
```

This means:

```txt
all states reachable from q using only λ-transitions
```

including `q` itself.

---

### Step 2: Add ordinary transitions

For every state `q` and input symbol `a`, define:

```txt
δ'(q, a) = λ-closure( δ( λ-closure(q), a ) )
```

Meaning:

```txt
1. From q, take any number of λ-moves.
2. Read one real input symbol a.
3. Then take any number of λ-moves again.
```

Those become direct non-λ transitions in the new NFA.

---

### Step 3: Update final states

A state `q` should be final in the new NFA if its λ-closure contains an old final state:

```txt
F' = { q | λ-closure(q) ∩ F ≠ ∅ }
```

Why?

Because from `q`, the old NFA could reach a final state without consuming input.

---

### Result

The new NFA has:

```txt
- same states
- same alphabet
- no λ-transitions
- same accepted language
```

So every NFA with λ-transitions can be converted to an equivalent NFA without λ-transitions.
