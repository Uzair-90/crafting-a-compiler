### Question No. 1
Assume the following text is presented to a C scanner:
```C
main(){
    const float payment = 384.00;
    float bal;
    int month = 0;
    bal=15000;
    while (bal>0){
        printf("Month: %2d Balance: %10.2f\n", month, bal);
        bal=bal-payment+0.015*bal;
        month=month+1;
    }
}
```


What token sequence is produced? For which tokens must extra information be returned in addition to the token code?

### Solution

```C
ID(main) ( ) {
    CONST FLOAT ID(payment) = FLOAT_LITERAL(384.00) ;
    FLOAT ID(bal) ;
    INT ID(month) = INT_LITERAL(0) ;
    ID(bal) = INT_LITERAL(15000) ;
    
    WHILE ( ID(bal) > INT_LITERAL(0) ) {
        ID(printf) ( STRING_LITERAL("Month: %2d Balance: %10.2f\n"),ID(month),ID(bal));
        ID(bal) = ID(bal) - ID(payment) + FLOAT_LITERAL(0.015) * ID(bal);
        ID(month) = ID(month) + INT_LITERAL(1) ;
    }
}
EOF
```

Extra information must be returned for:

```C
Identifiers:
main, payment, bal, month, printf

Numeric literals:
384.00, 0, 15000, 0.015, 1

String literal:
"Month: %2d Balance: %10.2f\n"
```

So the rule is: keywords, operators, and punctuation only need token codes; identifiers and literals need token code + extra value/lexeme information.
