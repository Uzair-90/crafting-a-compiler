### Question No. 14

Show that the set ```{[k ]k | k ≥ 1}``` is not regular. Hint: Show that no ﬁxed
number of FA states is suﬃcient to exactly match left and right brackets.

### Solution

### Exercise 14

Language:

```txt
L = { [^k ]^k | k ≥ 1 }
```

Meaning:

```txt
[]
[[]]
[[[]]]
[[[[]]]]
...
```

There are `k` left brackets `[` followed by `k` right brackets `]`.

### Claim

`L` is **not regular**.

### Proof using pumping lemma

Assume `L` is regular.

Then there is a pumping length `p`.

Choose the string:

```txt
s = [^p ]^p
```

So:

```txt
s = [[[...[[[]]]...]]]
```

with `p` left brackets and `p` right brackets.

By the pumping lemma, we can split:

```txt
s = xyz
```

such that:

```txt
|xy| ≤ p
|y| > 0
```

Since the first `p` characters are all `[`, both `x` and `y` are inside the left-bracket part.

So:

```txt
y = [^m
```

where `m > 0`.

Now pump down with `i = 0`:

```txt
xz
```

This removes some left brackets but keeps the same number of right brackets.

So we get:

```txt
[^p-m ]^p
```

Now the number of `[` and `]` is not equal.

Therefore:

```txt
xz ∉ L
```

This contradicts the pumping lemma.

### Conclusion

```txt
{ [^k ]^k | k ≥ 1 }
```

is **not regular** because a finite automaton cannot count and remember an arbitrary number of `[` symbols to match the same number of `]` symbols.
