### Question No. 20

Prove that the DFA constructed by MakeDeterministic in Section 3.8.2
is equivalent to the original NFA. To do so, you must show that an input
string can lead to a ﬁnal state in the NFA if, and only if, that same string
will lead to a ﬁnal state in the corresponding DFA.

### Solution

### Exercise 20

Claim: `MakeDeterministic` correctly converts an NFA into an equivalent DFA.

### Idea

The DFA state represents a **set of NFA states**.

So if the DFA is in state:

```txt
{q1, q3, q7}
```

that means:

```txt
After reading the same input, the NFA could be in q1, q3, or q7.
```

### Proof

For every input string `w`, prove:

```txt
DFA reaches state S after reading w
```

where:

```txt
S = all NFA states reachable after reading w
```

#### Base case

For empty string `λ`:

```txt
DFA starts at λ-closure({NFA start})
```

That is exactly the set of NFA states reachable before consuming input.

So the claim is true for `λ`.

#### Induction step

Assume true for string `w`.

Now read one more character `a`.

The NFA can move from every state in `S` using `a`, then take any λ-transitions.

So the next set is:

```txt
λ-closure(move(S, a))
```

That is exactly how `MakeDeterministic` creates the DFA transition.

So the claim is true for `wa`.

### Acceptance

The DFA accepts if its current set contains at least one NFA final state.

That matches the NFA rule:

```txt
NFA accepts if at least one possible path reaches a final state.
```

### Conclusion

Therefore, for every string:

```txt
DFA accepts w ⇔ NFA accepts w
```

So `MakeDeterministic` is correct.
