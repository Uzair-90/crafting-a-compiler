### Question No. 6

Write a regular expression that deﬁnes a C-like comment delimited by
/* and */. Individual *’s and /’s may appear in the comment body, but
the pair */ may not.


### Solution'

Exercise 6: **C-like block comment** starts with `/*`, ends with `*/`, and the body cannot contain `*/`.

A standard regular expression is:

```txt
/\*([^*]|\*+[^*/])*\*+/
```

In the book-style notation:

```txt
'/' '*' ( Not('*') | '*'* Not('*' | '/') )* '*'* '*''/'
```

Cleaner form:

```txt
'/' '*' ( Not('*') | '*' + Not('*' | '/') )* '*' + '/'
```

Meaning:

```txt
/\*        start comment
(...)*     body
\*+/       one or more stars followed by slash, the closing delimiter
```

Why this works:

```txt
/* hello */          valid
/**/                valid
/****/              valid
/* a / b * c */     valid
/* abc */ def */    first comment ends at first */
```

But inside the body, it never allows `*/`; that sequence is only used as the final closing delimiter.
