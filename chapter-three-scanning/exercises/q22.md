### Question No. 22

Write Lex regular expressions (using character classes if you wish) that
match the following sets of strings:
* The set of all unprintable ASCII characters (those before the blank
and the very last character)
* The string ["""] (that is, a left bracket, three double quotes, and a
right bracket)
* The string x12,345 (your solution should be far less than 12,345 char-
acters in length)

### Solution

Write Lex regular expressions for these strings/sets. The Lex section of the chapter explains character classes, quoted strings, escapes, and ranges. 

### (a) All unprintable ASCII characters

Unprintable ASCII characters are:

```txt
ASCII 0–31 and ASCII 127
```

In Lex:

```lex
[\000-\037\177]
```

Explanation:

```txt
\000-\037  matches ASCII control characters 0 to 31
\177       matches DEL, ASCII 127
```

---

### (b) The exact string `["""]`

That means:

```txt
[
"
"
"
]
```

A Lex regex:

```lex
\["""\]
```

or safer with escaped quotes:

```lex
\["\"\"\""\]
```

Simpler idea:

```txt
escape [ and ], and match three double quotes literally
```

---

### (c) The string `x12,345`

Since Lex has no exponent/repetition-count notation like `{12345}` in old-style Lex, use positive closure for digits:

```lex
x[0-9]+,[0-9]+
```

If the exact intended form is “x followed by digits, comma, digits,” this is best.

If it must match **exactly** `x12,345`, then:

```lex
x12,345
```

But the exercise says the solution should be much shorter than 12,345 characters, so the intended answer is:

```lex
x[0-9]+,[0-9]+
```
