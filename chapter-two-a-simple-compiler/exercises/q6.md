### Question No. 6

Variables are considered uninitialized after they are declared in some
programming languages. In ac a variable must be given a value in an
assignment statement before it can be correctly used in an expression or
print statement.
Suggest how to extend ac’s semantic analysis (Section 2.7) to detect
variables that are used before they are properly initialized.

### Solution

To detect variables used before initialization in ac, the semantic analyzer can extend each symbol table entry with an additional boolean attribute indicating whether the variable has been initialized. When a variable is declared, this attribute is set to false. Whenever the variable appears on the left-hand side of an assignment statement, the attribute is updated to true. During semantic analysis of expressions and print statements, the analyzer checks whether referenced variables have been initialized before use. If an identifier is encountered whose initialized attribute is false, a semantic error is reported. This approach allows the compiler to detect attempts to use variables before they have been assigned valid values.
