### Question No. 11

![image](../../images/fig-2.6.png) scans an input stream for an inum or fnum based on the regular
expressions for those patterns shown in Figure 2.3. The code in Figure 2.6
does not check for errors.
* Where could errors occur in Figure 2.6?
* For each error, what action would you take should the error occur?

### Solution

1. The scanner is only checking for '.' to identify int vs float and not checking the next immediate number after '.', what will happen incase of 123. or even 123abc? This may raise an error.

2. What could we do for each error if there are no digits after decimal we must return the number without the decimal as int like:

```bash

if decimal point is found:
    consume '.'
    if next character is not a digit:
        call LEXICALERROR()
if malformed number is found:
    call LEXICALERROR()

```
