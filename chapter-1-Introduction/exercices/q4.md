### **Question No. 4**
Cross-compilation assumes that a compiler for language X exists on some
machine. When the ﬁrst compiler for a new language is created, this
assumption does not hold. In this situation, a bootstrapping approach
can be taken. First, a subset of language X is chosen that is suﬃcient to
implement a simple compiler. Next, a simple compiler for the X subset
is written in any available language. This compiler must be correct, but
it should not be any more elaborate than is necessary, since it will soon
be discarded. Next, the subset compiler for X is rewritten in the X subset
and then compiled using the subset compiler previously created. Finally,
the X subset, and its compiler, can be enhanced until a complete compiler
for X, written in X, is available.
Assume you are bootstrapping C++ or Java (or some comparable lan-
guage). Outline a suitable subset language. What language features
must be in the language? What other features are desirable?

### **Solution**

A suitable bootstrap subset of C++ or Java must include basic data types (integers and booleans), variables and assignment, control flow constructs such as if-else and loops, function definitions and calls, arithmetic and comparison operators, and a minimal form of data structure such as arrays or pointers. These features are sufficient to implement a simple compiler because they allow representation of tokens, parsing logic, and code generation structures. Advanced features such as classes, templates, exception handling, and libraries are not required in the initial subset and can be added later once a self-hosting compiler is available. Desirable extensions include strings, structs, recursion, and basic file I/O to simplify compiler implementation.
