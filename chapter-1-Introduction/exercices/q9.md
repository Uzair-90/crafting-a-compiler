### **Question No. 9**

Many computer systems provide an interactive debugger (for example,
gdb or dbx) to assist users in diagnosing and correcting runtime errors.
Although a debugger is run long after a compiler has done its job, the
two tools still must cooperate. What information (beyond the transla-
tion of a program) must a compiler supply to support eﬀective runtime
debugging?

### **Solution**

To support effective runtime debugging, a compiler must provide more than just translated machine code. It must generate detailed debugging information including mappings between source code lines and machine instructions, symbol table entries for variable and function names, type information for correct interpretation of memory, and scope and lifetime data for variables. Additionally, it must provide stack frame and calling convention details so that the debugger can reconstruct function calls and execution state. This metadata allows debuggers such as GDB or DBX to present program execution in terms of the original source code rather than raw machine instructions, enabling meaningful inspection of variables and program flow during runtime.
