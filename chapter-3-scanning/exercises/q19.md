### Question No. 19

Let Rev be the operator that reverses the sequence of characters within
a string. For example, Rev(abc) = cba. Let R be any regular expression.
Rev(R) is the set of strings denoted by R, with each string reversed. Is
Rev(R) a regular set? Why or why not?

### Solution

Claim:

```txt
If R is regular, then Reverse(R) is also regular.
```

Here:

```txt
Reverse(R) = all strings of R written backward
```

Example:

```txt
R = ab*
Reverse(R) = b*a
```

### Proof idea

Start with an NFA for `R`.

Suppose the NFA is:

```txt
N = (Q, Σ, δ, q0, F)
```

Build a new NFA for the reversed language:

```txt
1. Reverse every transition.
2. Make the old final states into possible starting states.
3. Make the old start state the new final state.
4. If there are multiple old final states, create a new start state with λ-transitions to them.
```

So the new NFA is:

```txt
N' = (Q ∪ {qnew}, Σ, δ', qnew, {q0})
```

where:

```txt
qnew --λ--> each state in F
```

and every old transition:

```txt
p --a--> q
```

becomes:

```txt
q --a--> p
```

### Conclusion

Because NFAs recognize regular languages, and we can build an NFA for the reversed language:

```txt
Reverse(R) is regular.
```
