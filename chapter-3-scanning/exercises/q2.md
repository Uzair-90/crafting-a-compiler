### Question No. 2

How many lexical errors (if any) appear in the following C program?
How should each error be handled by the scanner?


```C
main(){
    if(1<2.)a=1.0else a=1.0e-n;
        subr(’aa’,"aaaaaa
    aaaaaa");
    /* That’s all
}

```

### Solution

There are **3 lexical errors**:

| Error                         | Why it is lexical                                                                      | Scanner handling                                                                                                                                                       |
| ----------------------------- | -------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `1.0e-n`                      | Invalid floating literal: after `e-`, C expects digits, not `n`.                       | Report **malformed numeric literal**. A reasonable recovery is to return `1.0` as a float token, then scan `e`, `-`, `n` separately, or delete the bad suffix.         |
| `"aaaaaa` followed by newline | Runaway string: a C string cannot contain an unescaped newline.                        | Report **unterminated string literal** at the newline. The scanner may return a repaired string token for `"aaaaaa"` or discard the bad string text up to the newline. |
| `/* That’s all`               | Unterminated comment: comment starts with `/*` but no closing `*/` appears before EOF. | Report **unterminated comment** and discard everything from `/*` to EOF.                                                                                               |

Important: `2.` is **not** a lexical error in C; C allows floating constants like `2.`. Also, `1.0else` is usually scanned as `FLOAT_LITERAL(1.0)` followed by keyword `else`; the missing separator causes a later syntax problem, not necessarily a scanner error. The book notes that C-like scanners must handle special lexical cases such as runaway strings/comments carefully, instead of treating them like ordinary illegal characters.
