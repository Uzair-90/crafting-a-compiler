### Exercise 14

The language consists of any subset of the optional symbols:

```txt
X1, X2, ..., Xn
```

where each symbol may appear either:

```txt
once
or
not at all
```

---

### Grammar

For symbols:

```txt
X1, X2, ..., Xn
```

a CFG can be written as:

```txt
S → O1 O2 O3 ... On

O1 → X1 | λ
O2 → X2 | λ
O3 → X3 | λ
...
On → Xn | λ
```

Each nonterminal `Oi` independently decides whether to include or omit `Xi`.

---

### Example (n = 3)

```txt
S  → O1 O2 O3

O1 → X1 | λ
O2 → X2 | λ
O3 → X3 | λ
```

This grammar generates:

```txt
λ
X1
X2
X3
X1 X2
X1 X3
X2 X3
X1 X2 X3
```

which are exactly all subsets of:

```txt
{ X1, X2, X3 }
```

---

### Size of the Grammar

The grammar contains:

```txt
1 start production
n optional productions
```

Therefore, the grammar size grows linearly with `n`:

```txt
O(n)
```

---

### If the Symbols Can Appear in Any Order

If the language allows arbitrary ordering of selected symbols, the grammar becomes much more complicated because it must generate every permutation of every subset.

For `n` symbols, the number of possible strings is:

```txt
Σ (n! / (n-k)!)
k=0..n
```

which grows exponentially.

Consequently, a simple linear-sized CFG is no longer sufficient.

---

### Key Lesson

This exercise demonstrates how optional constructs can be represented in a CFG using:

```txt
Xi | λ
```

and how a compact grammar can generate all subsets of a fixed set of symbols.
