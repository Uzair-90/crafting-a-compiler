### Exercise 5

Describe the language denoted by each grammar.

#### (a)

```txt
({ A, B, C }, { a, b, c }, ∅, A)
```

Here, the set of productions is empty:

```txt
P = ∅
```

So there is no rule that can rewrite the start symbol `A` into terminals.

Therefore, no strings can be derived.

```txt
L(G) = ∅
```

The language is the empty language.

---

#### (b)

```txt
({ A, B, C }, { a, b, c }, { A → B C }, A)
```

The only production is:

```txt
A → B C
```

But there are no productions for `B` or `C`.

So after deriving:

```txt
A ⇒ B C
```

the grammar gets stuck. It cannot produce a string made only of terminals.

Therefore:

```txt
L(G) = ∅
```

The language is the empty language.

---

#### (c)

```txt
({ A, B, C }, { a, b, c }, { A → A a, A → b }, A)
```

Productions:

```txt
A → A a
A → b
```

Starting from `A`, we can either stop with:

```txt
A ⇒ b
```

or apply `A → A a` repeatedly and finally use `A → b`.

Examples:

```txt
A ⇒ b
A ⇒ A a ⇒ b a
A ⇒ A a ⇒ A a a ⇒ b a a
```

So the language is:

```txt
L(G) = { b a^n | n ≥ 0 }
```

Equivalent regular expression:

```txt
ba*
```

---

#### (d)

```txt
({ A, B, C }, { a, b, c }, { A → B B, B → a, B → b, B → c }, A)
```

Productions:

```txt
A → B B
B → a
B → b
B → c
```

Since each `B` can become `a`, `b`, or `c`, `A` derives any string of exactly two symbols from `{a, b, c}`.

Examples:

```txt
aa
ab
ac
ba
bb
bc
ca
cb
cc
```

So the language is:

```txt
L(G) = { xy | x ∈ {a,b,c}, y ∈ {a,b,c} }
```

Equivalent regular expression:

```txt
(a | b | c)(a | b | c)
```
