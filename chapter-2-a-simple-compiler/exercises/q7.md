### Question No. 7

Implement semantic actions in the recursive-descent parser for ac to
construct ASTs using the design guidelines in Section 2.6.

Solution:

Semantic actions are added to the recursive-descent parser by having each parsing procedure construct and return AST nodes corresponding to the grammar constructs it recognizes. Declaration procedures create declaration nodes, statement procedures create assignment or print nodes, and expression procedures construct operator trees such as plus and minus nodes with operand children. Instead of producing a full parse tree containing every grammar symbol, the parser constructs a compact abstract syntax tree that retains only the essential semantic structure needed by later compiler phases such as semantic analysis, optimization, and code generation.


**AST node types for ac**

Based on Section 2.6, we need nodes for:

| Construct | AST Node |
|---|---|
| declarations | `DclNode` |
| assignment | `AssignNode` |
| print | `PrintNode` |
| plus | `PlusNode` |
| minus | `MinusNode` |
| identifiers | `IdNode` |
| integer literals | `IntNode` |
| float literals | `FloatNode` |
| statement sequences | `StmtListNode` |

**Helper constructors**

We assume constructors like:
```bash
MakeDclNode(type,id)
MakeAssignNode(id,expr)
MakePrintNode(id)
MakePlusNode(left,right)
MakeMinusNode(left,right)
MakeIdNode(name)
MakeIntNode(value)
MakeFloatNode(value)
MakeStmtListNode(stmt,rest)
```

**Recursive-descent parser with semantic actions**

MATCH
```C
procedure MATCH(expected)

    if lookahead.type = expected
    then
        lookahead ← SCANNER()

    else
        ERROR()

end
```

PROG
```bash
procedure PROG() returns AST

    dclsTree ← DCLS()
    stmtsTree ← STMTS()

    MATCH($)

    return MakeProgramNode(dclsTree, stmtsTree)

end
```
DCLS
```bash
procedure DCLS() returns AST

    if lookahead.type = floatdcl
       or lookahead.type = intdcl
    then

        dclTree ← DCL()
        restTree ← DCLS()

        return MakeDclListNode(dclTree, restTree)

    else
        return NULL

end
```
DCL
```bash
procedure DCL() returns AST

    if lookahead.type = floatdcl
    then
        MATCH(floatdcl)

        name ← lookahead.val
        MATCH(id)

        return MakeDclNode("float", name)

    else if lookahead.type = intdcl
    then
        MATCH(intdcl)

        name ← lookahead.val
        MATCH(id)

        return MakeDclNode("int", name)

    else
        ERROR()

end
```
STMTS
```bash
procedure STMTS() returns AST

    if lookahead.type = id
       or lookahead.type = print
    then

        stmtTree ← STMT()
        restTree ← STMTS()

        return MakeStmtListNode(stmtTree, restTree)

    else
        return NULL

end
```

STMT
```C
procedure STMT() returns AST

    if lookahead.type = id
    then

        name ← lookahead.val
        MATCH(id)

        MATCH(assign)

        valTree ← VAL()
        exprTree ← EXPR(valTree)

        return MakeAssignNode(name, exprTree)

    else if lookahead.type = print
    then

        MATCH(print)

        name ← lookahead.val
        MATCH(id)

        return MakePrintNode(name)

    else
        ERROR()

end
```
EXPR

```C
This builds left-associative expression trees.

procedure EXPR(leftTree) returns AST

    if lookahead.type = plus
    then

        MATCH(plus)

        rightTree ← VAL()

        newTree ← MakePlusNode(leftTree, rightTree)

        return EXPR(newTree)

    else if lookahead.type = minus
    then

        MATCH(minus)

        rightTree ← VAL()

        newTree ← MakeMinusNode(leftTree, rightTree)

        return EXPR(newTree)

    else
        return leftTree

end
```
VAL
```bash
procedure VAL() returns AST

    if lookahead.type = id
    then

        name ← lookahead.val
        MATCH(id)

        return MakeIdNode(name)

    else if lookahead.type = inum
    then

        value ← lookahead.val
        MATCH(inum)

        return MakeIntNode(value)

    else if lookahead.type = fnum
    then

        value ← lookahead.val
        MATCH(fnum)

        return MakeFloatNode(value)

    else
        ERROR()

end
```
