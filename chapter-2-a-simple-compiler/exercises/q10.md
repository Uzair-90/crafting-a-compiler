### Question No. 10

The code in Figure ![image](../../images/fig-2.10.png) examines an AST node to determine its eﬀect on
the symbol table. Explain why the order in which nodes are visited does
or does not matter with regard to symbol-table construction.

### Solution

For the original `ac` language, the order in which declaration nodes are visited generally does **not** matter for symbol-table construction, because all declarations appear before all executable statements in the grammar:

```text id="u16u0q"
Prog → Dcls Stmts $
```

The visitor method in Figure 2.10 simply processes declaration nodes and inserts identifiers into the symbol table:

```text id="fjlwmf"
EnterSymbol(name,type)
```

Since declarations are independent of one another, the compiler only needs to ensure that:

* each declared identifier is entered into the symbol table
* duplicate declarations are detected

For example:

```text id="vjlwm7"
intdcl a
floatdcl b
```

and

```text id="ifjlwm"
floatdcl b
intdcl a
```

produce essentially the same final symbol table:

| Identifier | Type    |
| ---------- | ------- |
| a          | integer |
| b          | float   |

Thus, for pure symbol-table construction, declaration visitation order is usually irrelevant as long as all declaration nodes are eventually visited.

However, order becomes important in situations involving:

* duplicate declarations
* scope rules
* use-before-declaration checking

For example, if the language were extended to allow declarations and statements to be interspersed:

```text id="jjlwmu"
print x
intdcl x
```

then semantic analysis would require top-to-bottom traversal because the symbol table must reflect what declarations have occurred *so far*. In such a language, visiting nodes out of order could incorrectly allow undeclared identifiers to appear valid.

Therefore, in the original `ac` language, node visitation order does not significantly affect symbol-table construction because declarations are grouped at the beginning of the program. But in more realistic languages with scopes, nested declarations, or interspersed statements, traversal order becomes semantically important.
