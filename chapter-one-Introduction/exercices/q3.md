**Solution**

To create a compiler for language X on system Y starting from system Z, you first write or obtain a cross-compiler that runs on system Z but generates code for system Y. This cross-compiler is then used to compile the source code of a new compiler written in language X itself. The result is an executable compiler for system Y that can now run on Y and generate code for Y, completing the bootstrap process. If system Y is bare (with no operating system or tools), the process becomes more difficult because there is no runtime environment, libraries, or system services available. In such cases, a minimal toolchain or runtime must first be created, often requiring manual assembly code or firmware-level programming before a full compiler system can be brought up.
